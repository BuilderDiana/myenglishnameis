# 🚀 明日计划：部署、微信生态接入与数据持久化

这是我们针对 **MyEnglishNameIs** 下一阶段（商业化与上线）的详细待办清单。

## 1. 部署方案选择 (Deployment)

既然你有 **AWS Credits**，且希望展示 **“工程思维”**，那么使用 AWS 是非常明智的选择：

- [ ] **方案 A：AWS Amplify / App Runner (推荐)**
  - 优点：对 Next.js 支持良好，Credits 抵扣成本，展示你对主流云厂商的掌控力。
  - 缺点：配置比 Vercel 略复杂，需要处理 IAM 权限。
- [ ] **方案 B：AWS EC2 + Docker (硬核方案)**
  - 优点：完全掌控环境，方便配置固定 IP（微信支付回调非常需要）。
  - 缺点：运维成本高。

## 2. 微信生态接入 (WeChat Integration)

这是商业化的核心，难度较高，需要按部就班：

- [ ] **微信公众号/服务号申请**：获取 `AppID` 和 `AppSecret`。
- [ ] **微信登录 (OAuth2.0)**：
  - 实现用户身份识别，为保存名字记录做准备。
  - 处理微信浏览器内的“静默授权”。
- [ ] **微信支付 (WeChat Pay)**：
  - 申请商户号。
  - 实现支付回调逻辑（Webhook），确保支付成功后解锁高级功能。

## 3. 数据库与数据持久化 (Database)

为了保存用户的生成记录和支付状态：

- [ ] **数据库选型**：
  - **Supabase (PostgreSQL)**：开发速度最快，自带 Auth 和 API。
  - **MongoDB Atlas**：适合存储非结构化的 AI 生成结果。
- [ ] **Schema 设计**：
  - `users` 表：存储微信 OpenID、昵称、会员状态。
  - `generation_records` 表：存储用户的输入参数和 AI 返回的名字列表。
  - `orders` 表：存储支付订单状态。

## 4. 商业化功能闭环 (Monetization)

- [ ] **付费墙逻辑**：实现我们今天设计的“解锁更多名字”和“深度报告”的权限控制。
- [ ] **用户中心**：增加“我的收藏”或“历史记录”页面。
